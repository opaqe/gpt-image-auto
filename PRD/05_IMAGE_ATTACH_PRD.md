## 1. 기능 개요

### 한 줄 요약
각 프롬프트 행에 참고 이미지를 첨부하면, 실행 시 ChatGPT에 텍스트 + 이미지를 함께 전송한다.

### 해결하는 문제
현재는 텍스트 프롬프트만 ChatGPT에 전송할 수 있다.  
ChatGPT는 이미지를 함께 받으면 스타일·구도·색감을 더 정확하게 반영한 이미지를 생성한다.  
참고 이미지 없이 텍스트만으로 원하는 결과를 얻으려면 프롬프트가 매우 길고 정교해야 한다.

### 핵심 가치
- 참고 이미지 + 텍스트 조합 → ChatGPT의 시각적 이해 활용
- 10개 프롬프트 각각에 서로 다른 참고 이미지 설정 가능
- 기존 자동화 흐름 (순차 실행, 완료 감지, 전역 지침, 체크박스) 완전히 유지

## 2. 사용자

### 주요 사용자
- **누구**: ChatGPT로 배치 이미지 생성 작업을 하는 디자이너/크리에이터
- **상황**: 레퍼런스 기반 카드뉴스, SNS 시리즈, 제품 이미지 시리즈 제작
- **목표**: 기존 이미지 스타일을 유지하면서 새로운 변형 이미지를 대량 생성

### 사용자 시나리오
1. 사용자가 팝업을 열고 프롬프트 #1 행의 📎 버튼을 클릭한다
2. 파일 선택 대화상자에서 참고 이미지(png/jpg/webp)를 고른다
3. 행에 소형 썸네일이 표시됨 — ✕ 버튼으로 제거 가능
4. "실행" 버튼을 누르면 프롬프트 #1 실행 시 ChatGPT에 이미지 + 텍스트가 함께 전송됨
5. 이미지 없는 다른 행은 기존과 동일하게 텍스트만 전송

## 3. 핵심 기능

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|----------|--------|
| 📎 첨부 버튼 | 각 프롬프트 행 우측에 파일 선택 버튼 | P1 | 간단 |
| 썸네일 미리보기 | 선택 이미지 32px 썸네일 + ✕ 제거 버튼 | P1 | 간단 |

| 업로드 완료 대기 | 업로드 완료 후 텍스트 입력 → 전송 순서 보장 | P1 | 보통 |

## 4. 사용자 흐름 (User Flow)

### 첨부 흐름
```
[📎 클릭] → [OS 파일 선택창] → [이미지 선택] → [썸네일 표시] → [✕ 제거 가능]
```

### 실행 흐름 (이미지 첨부 프롬프트)
```
processNext()
  └─ 이미지 있음?
      ├─ YES → attachImageToChat(base64)
      │           └─ [파일 입력 주입] → [업로드 완료 대기] → typePrompt() → clickSend()
      └─ NO  → typePrompt() → clickSend() (기존과 동일)
```

### 상세 흐름
1. **팝업 UI**: `<input type="file" accept="image/*">` (hidden) — 📎 버튼이 클릭 트리거
2. **메모리 저장**: `FileReader.readAsDataURL()` → `attachedImages[i]` (세션 동안 유지)
3. **실행 시 전달**: `START_GENERATION` 메시지의 `images` 필드에 base64 배열 포함
4. **content.js 업로드**:
   - base64 → `fetch()` → Blob → `File` 객체 생성
   - ChatGPT의 `input[type="file"]`에 `DataTransfer`로 파일 주입
   - `change` 이벤트 디스패치
   - 업로드 완료 신호 대기 (ChatGPT 첨부 UI 썸네일 등장 또는 고정 딜레이)
5. **이후 흐름**: 기존 `typePrompt()` → `clickSend()` → `waitForCompletion()` 그대로

## 5. 성공 기준

- [ ] 프롬프트 행에서 파일을 선택하면 썸네일이 즉시 표시된다
- [ ] ✕ 버튼으로 첨부 이미지를 제거할 수 있다
- [ ] 이미지 첨부된 프롬프트 실행 시 ChatGPT 입력창에 이미지가 나타난다
- [ ] 텍스트 전송 전 이미지 업로드가 완료되어 있다
- [ ] 이미지 없는 프롬프트는 텍스트만 전송한다 (기존 동작 유지)
- [ ] 전역 지침, 체크박스 선택, 드래그 순서 변경이 이미지 첨부와 함께 정상 작동한다

## 6. 안 만드는 것 (Out of Scope)

- **이미지 저장 (세션 간 유지)** — 이유: 팝업 닫으면 리셋. IndexedDB 구현 복잡도 대비 효용 낮음
- **다중 이미지 첨부 (1행에 여러 장)** — 이유: ChatGPT 동작 예측 어려움, 1장으로 충분
- **URL로 이미지 입력** — 이유: CORS·인증 문제 발생 가능성
- **드래그 앤 드롭** — 이유: 팝업 창 크기가 작아 드래그 영역 협소
